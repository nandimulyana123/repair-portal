Manual Book Portal Pengajuan Perbaikan

Versi dokumen: 14 April 2026

Dokumen ini dipakai untuk panduan operasional aplikasi Portal Pengajuan Perbaikan (mode localhost) oleh Admin dan User.

1. Persiapan & Menjalankan Aplikasi

  1. Buka folder aplikasi repair-portal.
  2. Jalankan perintah py backend.py.
  3. Buka browser ke http://localhost:8000.

Akun awal:

  • Admin: admin / Admin123!
  • User: user / User123!

2. Alur Kerja User

  1. Login sebagai user.
  2. Buka tab Pengajuan.
  3. Isi Judul, Kategori, Lokasi, Unit, Prioritas, dan Deskripsi.
  4. Klik Kirim Pengajuan.
  5. Pantau status di tabel pengajuan pribadi.

3. Alur Kerja Admin

  1. Login sebagai admin.
  2. Buka tab Admin Panel.
  3. Kelola status pengajuan: Diajukan, Diproses, Selesai, Ditolak.
  4. Isi Alasan Proses saat status Diproses.
  5. Pakai filter status/bulan/pencarian untuk monitoring.

4. Manajemen Pengguna (Admin)

Tambah User

  • Isi Username, Nama, Role, Password.
  • Password wajib: 8+ karakter, huruf besar, huruf kecil, angka, simbol.

Kelola User

  • Ubah Password: input manual sesuai kebijakan.
  • Password Acak: sistem generate password kuat dan tampil sekali.
  • Aktifkan/Nonaktifkan: kontrol akses login user.
  • Hapus: hanya bisa jika user tidak punya riwayat pengajuan.

5. Rekap & Laporan

  • Tab Rekap Bulanan untuk ringkasan berdasarkan bulan.
  • Admin bisa klik Unduh PDF untuk laporan bulanan.
  • Pengaturan kop surat dan penandatangan di menu admin.

6. Audit Log

  • Menampilkan aktivitas penting admin (buat user, reset password, status user, update request, reset DB).
  • Bisa difilter berdasarkan Admin, Aksi, dan Tanggal.
  • Mendukung pagination (halaman).
  • Bisa diekspor ke CSV dan PDF.

7. Backup & Restore Data

File database utama adalah repair-portal.db.

  • Backup: salin file repair-portal.db ke folder cadangan.
  • Restore: hentikan server, timpa repair-portal.db dengan file backup, lalu jalankan server lagi.

8. Troubleshooting Cepat

  • Tidak bisa login: cek username/password dan status user (aktif/nonaktif).
  • Halaman tidak terbuka: pastikan py backend.py sedang berjalan.
  • PDF tidak keluar: refresh halaman, lalu coba ulang export.
  • Data tidak sesuai: cek filter bulan/status/pencarian pada tabel.
